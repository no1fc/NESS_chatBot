# Next.js 16 + SQLite Ubuntu 서버 배포 가이드 (표준 방식)

본 문서는 Ubuntu 24.04 LTS 서버 환경에서 소스 코드를 직접 업로드하여 빌드하고 실행하는 표준 배포 방식에 대한 가이드입니다.

---

## 🏗 플랫폼 요약 및 아키텍처

* **OS**: Ubuntu 24.04 LTS
* **기존 환경**: Tomcat 11, Java 17, MSSQL, Nginx, FTP(vsftpd), Redis 설치 완료
* **배포 스택**: Next.js 16 (React 19), SQLite (`better-sqlite3`)
* **핵심 전략**: 소스 전체 업로드 후 서버에서 직접 의존성 설치 및 빌드 수행

---

## 1단계: 서버 환경 구성 (Node.js & 빌드 도구)

1. **Node.js 20 (LTS) 설치**

```bash
sudo apt update
sudo apt install -y curl build-essential

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

1. **PM2 설치**

```bash
sudo npm install -g pm2
```

---

## 2단계: 소스 코드 업로드

로컬 PC에서 다음 항목들을 제외한 **전체 소스 코드**를 서버의 `/opt/NESSChatBot/next-app/` 경로로 업로드합니다.

**제외 항목 (업로드 금지):**
* `node_modules/`
* `.next/`
* `.git/`

**필수 포함 항목:**
* `src/`, `public/`, `data/`
* `package.json`, `package-lock.json`
* `next.config.ts`, `tsconfig.json` 등 모든 설정 파일
* `.env.local`
* `국취유형별자가진단.pdf`

---

## 3단계: 서버 측 빌드 및 권한 설정

1. **배포 경로 이동 및 의존성 설치**

```bash
cd /opt/NESSChatBot/next-app
npm install
```

1. **프로젝트 빌드**

```bash
npm run build
```

1. **SQLite 쓰기 권한 설정 (필수)**

```bash
chmod 775 /opt/NESSChatBot/next-app/data
chmod 664 /opt/NESSChatBot/next-app/data/mainDB.db
```

---

## 4단계: PM2로 서비스 실행

```bash
cd /opt/NESSChatBot/next-app

# PM2를 통해 npm start 명령어로 실행
PORT=3000 pm2 start npm --name "ness-chatbot" -- start

# 자동 재시작 설정
pm2 startup
# (출력되는 명령어 복사/붙여넣기 실행)
pm2 save
```

---

## 5단계: Nginx 리버스 프록시 설정 (포트 81 예시)

`http://jobmoaa.mycafe24.com:81`로 접속할 수 있도록 설정하는 방법입니다.

1. **설정 파일 생성**: `sudo nano /etc/nginx/sites-available/ness-chatbot`
2. **내용 입력**:

```nginx
server {
    listen 81;
    server_name jobmoaa.mycafe24.com;

    client_max_body_size 50M;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

1. **활성화 및 재시작**

```bash
sudo ln -s /etc/nginx/sites-available/ness-chatbot /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

---

## 6단계: 방화벽(iptables) 및 로그 확인

1. **포트 81 개방 (필수)**

```bash
sudo iptables -I INPUT -p tcp --dport 81 -j ACCEPT
sudo apt install -y iptables-persistent
sudo netfilter-persistent save
```

1. **로그 확인**

```bash
pm2 logs ness-chatbot
```
