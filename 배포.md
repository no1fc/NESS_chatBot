# Next.js 16 + SQLite Ubuntu 서버 배포 가이드 (표준 방식)

본 문서는 Ubuntu 24.04 LTS 서버 환경에서 소스 코드를 직접 업로드하여 빌드하고 실행하는 표준 배포 방식에 대한 가이드입니다.

---

## 🏗 플랫폼 요약 및 아키텍처

* **OS**: Ubuntu 24.04 LTS
* **기존 환경**: Tomcat 11, Java 17, MSSQL, Nginx, FTP(vsftpd), Redis 설치 완료
* **배포 스택**: Next.js 16 (React 19), SQLite (`better-sqlite3`)
* **핵심 전략**: 소스 전체 업로드 후 서버에서 직접 의존성 설치 및 빌드 수행

---

## 1단계: 서버 환경 구성 (Node.js & 빌드 도구)

1. **Node.js 20 (LTS) 설치**

```bash
sudo apt update
sudo apt install -y curl build-essential

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

1. **PM2 설치**

```bash
sudo npm install -g pm2
```

---

## 2단계: 소스 코드 업로드

로컬 PC에서 다음 항목들을 제외한 **전체 소스 코드**를 서버의 `/opt/NESSChatBot/next-app/` 경로로 업로드합니다.

**제외 항목 (업로드 금지):**

* `node_modules/`
* `.next/`
* `.git/`

**필수 포함 항목:**

* `src/`, `public/`, `data/`
* `package.json`, `package-lock.json`
* `next.config.ts`, `tsconfig.json` 등 모든 설정 파일
* `.env.local`
* `국취유형별자가진단.pdf`

---

## 3단계: 서버 측 빌드 및 권한 설정

1. **배포 경로 이동 및 의존성 설치**

```bash
cd /opt/NESSChatBot/next-app
npm install
```

1. **프로젝트 빌드**

```bash
npm run build
```

1. **SQLite 쓰기 권한 설정 (필수)**

```bash
chmod 775 /opt/NESSChatBot/next-app/data
chmod 664 /opt/NESSChatBot/next-app/data/mainDB.db
```

---

## 4단계: PM2로 서비스 실행

```bash
cd /opt/NESSChatBot/next-app

# PM2를 통해 npm start 명령어로 실행
PORT=3000 pm2 start npm --name "ness-chatbot" -- start

# 자동 재시작 설정
pm2 startup
# (출력되는 명령어 복사/붙여넣기 실행)
pm2 save
```

---

## 5단계: 서비스 접속 포트 개방 (Iptables) 및 보안 설정

현재 Nginx는 포트 80 접속 시 포트 8088로 자동 리다이렉트 되도록 글로벌 구성을 사용하고 있습니다. 따라서 **NESS 챗봇 서비스는 Nginx 없이 포트 3000을 직접 외부로 개방**하여 사용합니다 (`http://jobmoaa.mycafe24.com:3000`).

1. **포트 3000 개방 (필수)**

```bash
sudo iptables -I INPUT -p tcp --dport 3000 -j ACCEPT
sudo apt install -y iptables-persistent
sudo netfilter-persistent save
```

### ⚠️ 포트 직접 개방에 따른 보안 조치사항

Nginx와 같은 리버스 프록시(Reverse Proxy)를 거치지 않고 Node.js 서버(포트 3000)가 바로 외부망에 노출되기 때문에 보안 관리가 필요합니다.

1. **DDoS 및 무차별 대입(Brute-force) 공격 방어**
   * 리버스 프록시가 제공해주는 요청 버퍼링 기능을 사용할 수 없으므로, Iptables를 이용해 동일 IP의 과도한 데이터 연결을 제한(Rate Limiting)하는 것이 안전합니다.
   * 예: 단일 IP 기준 동시 접속 수를 20개로 제한하기

     ```bash
     sudo iptables -A INPUT -p tcp --dport 3000 -m connlimit --connlimit-above 20 -j REJECT
     sudo netfilter-persistent save
     ```

2. **HTTPS(SSL/TLS) 연동의 한계**
   * 포트 3000번 접속 시 기본적으로 평문 전송인 `http`로 접속됩니다. 민감한 정보(로그인 등)를 다루므로, 향후 서비스 본격 오픈 시 도메인에 Cloudflare와 같은 프록시를 씌워 외부 통신 구간 HTTPS를 강제 확보하거나, 추후 Nginx를 통하도록 구성을 변경하는 것을 권장합니다.
3. **특정 IP 대역 허용 (선택적용)**
   * 만약 대국민 서비스가 아닌 관리자 전용 또는 지정된 기관용 서비스라면, `0.0.0.0/0` 전체 허용보다는 허용된 특정 IP망에서만 3000번 포트로 들어올 수 있게 등록하는 방식이 가장 안전합니다.

## 6단계: 서비스 접속 및 정상 작동 확인

방화벽 설정이 완료되면 아래 주소로 접속하여 서비스가 정상 작동하는지 확인합니다.

1. **사용자 챗봇 홈 페이지**: `http://jobmoaa.mycafe24.com:3000/`
2. **관리자 로그인 페이지**: `http://jobmoaa.mycafe24.com:3000/admin/login`
   * 브라우저 주소창에는 반드시 `/admin/login`을 입력해야 합니다. (실수로 API 주소 오류 접근 요청 시에도 자동으로 리다이렉트 되도록 안전 조치 적용됨)

3. **초기 관리자 계정**:
   * 기본값(`nessadmin` / `ness1234!`) 또는 `.env.local`에 별도 설정한 `ADMIN_ID`, `ADMIN_PASSWORD` 값으로 로그인 가능합니다.

## 7단계: 서버 구동 로그 확인 및 재배포/재실행 가이드

소스 코드를 수정한 후 서버에 재반영할 경우 다음 프로세스로 진행합니다.

1. **에러 발생 및 실시간 로그 확인**

```bash
# 구동 중인 PM2 로그 보기
pm2 logs ness-chatbot
```

1. **소스 코드 업데이트 및 서버 재빌드 (재배포)**

```bash
# 1) 로컬 PC에서 변경된 파일 업로드 후 배포 디렉토리로 이동
cd /opt/NESSChatBot/next-app

# 2) 만약 모듈(패키지) 추가가 있었다면 의존성 재설치
npm install

# 3) Next.js 프로젝트 재빌드 (새로운 내용 반영)
npm run build
```

1. **서비스 재실행 (최소 중단 재시작)**

```bash
# 4) PM2 프로세스 재시작 (reload 사용 시 다운타임 최소화)
pm2 reload ness-chatbot
# (또는 pm2 restart ness-chatbot 명령어를 사용 가능)

# 5) 변경된 PM2 구동 상태를 저장하여 재부팅 시에도 구동되게 유지
pm2 save
```
